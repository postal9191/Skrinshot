# Skrinshot 📸

Desktop-приложение для скриншотов и записи экрана с загрузкой файлов на локальный сервер.

## Возможности

- 📷 **Скриншоты**
  - Всего экрана
  - Выбранной области
  - Активного окна
  - Поддержка нескольких мониторов
  - Корректная работа с DPI scaling

- 🎨 **Редактор скриншотов**
  - Стрелки
  - Линии
  - Прямоугольники и эллипсы
  - Текст
  - Маркер/подсветка
  - Размытие областей

- 🎥 **Запись экрана**
  - Запись всего экрана или области
  - Настройка FPS (15/30/60)
  - Настройка битрейта
  - Сохранение в MP4/WebM

- ☁️ **Загрузка на сервер**
  - HTTP API для загрузки
  - Поддержка Bearer Token
  - Очередь файлов при недоступности сервера
  - Автоматическая повторная отправка

- 📋 **История файлов**
  - Последние 100 файлов
  - Статус загрузки
  - Быстрый доступ к файлам

## Установка

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск в режиме разработки

```bash
# Запуск приложения
npm run dev

# Запуск сервера (в отдельном терминале)
npm run server:dev
```

### 3. Сборка приложения

```bash
# Сборка
npm run build

# Создание установщика
npm run electron:build
```

## Горячие клавиши

- `PrintScreen` — Скриншот всего экрана
- `Ctrl+PrintScreen` — Скриншот выбранной области
- `Shift+PrintScreen` — Запись экрана

## Настройка сервера

Сервер запускается на порту 8080 по умолчанию.

### Переменные окружения

- `PORT` — порт сервера (по умолчанию 8080)

### API Endpoints

- `GET /api/health` — проверка состояния
- `POST /api/upload/image` — загрузка изображения
- `POST /api/upload/video` — загрузка видео
- `GET /api/files/:id` — информация о файле

### Пример ответа API

```json
{
  "success": true,
  "fileId": "abc123",
  "fileName": "screenshot_2026-03-13_10-25-11.png",
  "url": "http://192.168.1.100:8080/files/abc123",
  "size": 245821
}
```

## Технологии

- **Electron** — кроссплатформенное desktop-приложение
- **React 18** — UI компоненты
- **TypeScript** — типизация
- **Express** — локальный сервер
- **Multer** — загрузка файлов
- **Vite** — сборка frontend

## 🧪 Тестирование после правок

После каждого редактирования кода **обязательно** выполняйте следующие шаги:

### 1. Сборка проекта
```bash
npm run build
```

### 2. Запуск и тестирование функций
```bash
# Запуск portable версии для тестирования
cd release\win-unpacked
start Skrinshot.exe
```

### 3. Проверка функций
- [ ] **F1 / PrintScreen** — полный скриншот экрана (открывается в редакторе, не сохраняется автоматически)
- [ ] **Ctrl+F1 / Ctrl+PrintScreen** — частичный скриншот области (чёрный квадрат?)
- [ ] **Редактор** — рисование стрелок, линий, текста
- [ ] **Копировать** — скриншот копируется в буфер обмена
- [ ] **Сохранить** — скриншот сохраняется в папку "Изображения"

### 4. Проверка логов
Откройте DevTools в окне приложения (`Ctrl+Shift+I`) и проверьте консоль на ошибки.

### 5. Финальная сборка релиза
После всех правок:
```bash
npm run electron:build
```

Готовая версия находится в `release\win-unpacked\` и `release\Skrinshot Setup 1.0.0.exe`
